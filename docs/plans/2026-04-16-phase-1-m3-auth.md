## Task 3.2: Field redaction respects userType

**Files:**
- Modify: `src/lib/rbac.ts`
- Modify: `src/lib/rbac.test.ts` (create if absent)

### Step 1: Extend `canAccessSensitiveFields` and `redactSensitiveFields`

New signature: `(userType, role)` or take the full user context.

```ts
export type UserCtx = {
  userType: 'internal' | 'external';
  role: 'admin' | 'member' | 'viewer' | null;
};

export function canAccessSensitiveFields(user: UserCtx): boolean {
  if (user.userType === 'external') return false;
  return user.role === 'admin' || user.role === 'member';
}

export function redactSensitiveFields<T extends Record<string, unknown>>(
  data: T,
  user: UserCtx,
): T {
  if (canAccessSensitiveFields(user)) return data;
  const redacted: T = { ...data };
  const sensitiveKeys: Array<keyof T | string> = [
    'bankingDetails', 'contractValue', 'contractTerms', 'contractDocuments',
    // External users also lose: contacts, maintenance fee
  ];
  if (user.userType === 'external') {
    sensitiveKeys.push(
      'keyContactName', 'keyContactEmail', 'financeContact',
      'maintenanceFee',
    );
  }
  for (const k of sensitiveKeys) {
    if (k in redacted) (redacted as any)[k] = null;
  }
  return redacted;
}
```

### Step 2: Update callers

Grep for existing `redactSensitiveFields(data, role)` call sites and update them to pass the full user context (userType + role). Keep backwards compatibility ONLY if trivial; otherwise update the call sites as a single atomic change.

### Step 3: Write unit tests for every combination

- `internal admin` → no redaction
- `internal member` → no redaction (currently)
- `internal viewer` → banking/contract redacted, contacts kept
- `external` (regardless of role, per invariant) → banking/contract redacted, contacts + maintenance also redacted
