# Phase 04.1 Plan 01: Schema Extension + Field Mapper Bug Fixes + Unit Tests Summary

**One-liner:** Schema extended with 4 location fields, kioskConfigGroups/appSettings tables, and 5 field-mapper bugs fixed (kioskId derivation, Assets mapping, notes pollution, key contact routing, location merge dedup) with 18 passing Vitest unit tests.

## What Was Built

### Task 1 — Schema Extension

Extended `src/db/schema.ts` with:

1. **`kioskConfigGroups` table** — new entity for content-based kiosk groupings (name unique, timestamps). Defined before `kiosks` table to satisfy FK forward reference.

2. **`appSettings` table** — key-value store for persistent board ID settings (D-13). Primary key is `key` text column.

3. **4 new columns on `locations` table** — `region`, `locationGroup`, `internalPoc`, `status` (all nullable text, per D-22/MIGR-11).

4. **`kioskConfigGroupId` FK on `kiosks` table** — nullable UUID referencing `kioskConfigGroups.id` (D-02).

All changes pushed to the Supabase database via `npx drizzle-kit push` (not migrate — session-mode pooler constraint per STATE.md).

### Task 2 — Field Mapper Bug Fixes + Unit Tests

**5 bugs fixed in `src/lib/field-mapper.ts`:**

| Bug | ID | Was | Now |
|-----|----|-----|-----|
| Assets mapping | MIGR-05 | `location.hardwareAssets` | `kiosk.hardwareSerialNumber` |
| KioskId source | MIGR-04 | `item.group?.title` (board group name) | Region column value from `column_values` |
| Notes pollution | MIGR-07 | All non-kiosk columns → notes | Only truly unmapped (no mapping / status="unmapped") |
| Key Contact | MIGR-08 | `location.keyContactName` | `location.internalPoc` |
| Location Group | D-22 | `kiosk.regionGroup` | `location.locationGroup` |

**New exports added:**
- `MergeConflict` type (moved before `ImportPreview` in types section to avoid forward reference)
- `normaliseLocationName(rawName)` — strips `[outlet_code]` suffix (MIGR-09)
- `detectMergeConflicts(locationGroups)` — field-level conflict detection for dry-run (D-10)
- `ImportPreview.conflicts: MergeConflict[]` — added to interface

**`src/app/(app)/settings/data-import/actions.ts` updates:**
- Imports `normaliseLocationName`, `detectMergeConflicts`
- `runDryImport` builds `locationGroups` map by normalised name, calls `detectMergeConflicts`, includes `conflicts` in returned `ImportPreview`
- `runFullImport` uses `normaliseLocationName(extractLocationName(item))` as dedup key so same-name hotels with different outlet code suffixes merge into one location
- `kioskItemMeta` entries use normalised hotel name for assignment lookup

**Unit tests: `src/lib/field-mapper.test.ts`** — 18 tests covering all 8 required behaviors:
- MIGR-05: Assets → hardwareSerialNumber (2 tests)
- MIGR-04: KioskId from Region column value (3 tests)
- MIGR-07: Notes not polluted by location fields (1 test)
- MIGR-08: Key Contact → internalPoc (2 tests)
- MIGR-06: CMS Config returns label string (1 test)
- MIGR-09: normaliseLocationName (5 tests)
- D-10: detectMergeConflicts conflict case + no-conflict case + single row + name field skip (4 tests)
