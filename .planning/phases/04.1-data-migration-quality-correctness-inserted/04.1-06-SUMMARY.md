## Decisions Made
- Used `fetchAllItems` directly in `importConfigGroups` rather than `exploreBoardColumns` — the plan described `exploreBoardColumns` as returning `{ columns, items }` but the actual implementation returns `{ success, columns, mappings }` with no items. `fetchAllItems` is the correct async generator for item fetching.
- Matched `kiosk.regionGroup` to group name for FK assignment, matching the plan's name-match strategy for the underdocumented board structure.
- No row actions on ConfigGroupsTable — config groups are read-only/import-only per UI-SPEC.
