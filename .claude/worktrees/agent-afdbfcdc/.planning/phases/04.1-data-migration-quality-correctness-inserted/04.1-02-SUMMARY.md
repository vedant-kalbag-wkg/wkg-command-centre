## What Was Built

### Task 1: Board ID Persistence Server Actions

- `loadBoardIds()` — reads saved board IDs from `appSettings` table for keys `boardId.kiosks`, `boardId.hotels`, `boardId.kioskConfigGroups`; returns empty strings gracefully on error (table may not exist yet from Plan 01)
- `saveBoardIds()` — upserts non-empty board IDs via `onConflictDoUpdate`; admin-only
- Extended `runFullImport()` with `conflictResolutions?: Record<string, Record<string, string>>` parameter; applies chosen values during the location deduplication step
- Added `appSettings` table to `src/db/schema.ts` (key-value store, text primary key)
- Added `MergeConflict` type and `conflicts?: MergeConflict[]` field to `ImportPreview` in `src/lib/field-mapper.ts`

### Task 2: Extended Data Import UI

- **Three board ID inputs** in connect step: Assets/Kiosks (`boardId-kiosks`), Hotels/Locations (`boardId-hotels`), Kiosk Config Groups (`boardId-configGroups`)
- **On mount** calls `loadBoardIds()` to pre-populate inputs; saves to `savedBoardIdsRef` for badge comparison
- **Debounced auto-save** (500ms) on each input change via `saveBoardIds()`; Azure "Saved" badge appears when current value matches persisted
- **ImportSubStepIndicator** component showing "Step 1 of 3 — Kiosks", "Step 2 of 3 — Hotels", "Step 3 of 3 — Config Groups" with Azure active state and checkmarks for completed steps
- **Primary CTA**: "Run Import" button (Azure `bg-[#00A6D3]`) — disabled unless kiosks board ID is filled
- **Individual import buttons**: "Import Kiosks", "Import Hotels", "Import Config Groups" (outline variant)
- **ConflictResolutionCard** component with Azure-tinted border (`border-[#00A6D380]`), per-field radio groups with Azure accent on selected value, conflict prompt per UI-SPEC copy
- **Import blocking**: "Run Import" disabled with copy "Resolve all field conflicts before importing. {N} conflict(s) remaining." until all conflict fields are resolved
- **Error copy** per UI-SPEC: board ID missing, import failed mid-run (step number interpolated)
