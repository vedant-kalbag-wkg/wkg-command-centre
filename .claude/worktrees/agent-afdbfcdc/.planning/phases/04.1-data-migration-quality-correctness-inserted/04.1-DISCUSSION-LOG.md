## Kiosk Config Groups Entity

### What does the Kiosk Groups board contain?

| Option | Description | Selected |
|--------|-------------|----------|
| Regional grouping | Groups are regions (e.g. London, Manchester) — each group contains multiple kiosks | |
| Hotel chain grouping | Groups are hotel chains/brands (e.g. Marriott, Hilton) | |

**User's choice:** Other — "They are grouped based on the content that is available on the kiosk"

### How should Kiosk Groups relate to kiosks?

| Option | Description | Selected |
|--------|-------------|----------|

| Many-to-many | Kiosks can belong to multiple groups | |
| Group is metadata only | Standalone table with no FK relationship | |

**User's choice:** One group for now, multiple hierarchical groups in the future.

**User's choice:** Other — Table with group name, product availability columns, clickable hotel count (filtered locations page), clickable kiosk count (filtered kiosks page). Groups board is source of truth for total offerings during import.

| Option | Description | Selected |

| Option | Description | Selected |
