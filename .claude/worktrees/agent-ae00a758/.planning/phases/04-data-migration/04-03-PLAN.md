**File 1: `src/app/(app)/locations/[id]/products/actions.ts`**

Server actions for product configuration:

**1. `listLocationProducts(locationId: string)`:**
- Joins `locationProducts` with `products` and `providers`
- Returns array of: `{ id, productName, productId, providerName, providerId, availability, commissionTiers }`
- Ordered by product name alphabetically

**2. `listAllProducts()`:**
- Returns all products from `products` table
- Used for "Add Product" dropdown

**3. `listAllProviders()`:**
- Returns all providers from `providers` table
- Used for provider assignment dropdown

**4. `updateLocationProduct(id: string, data: { availability?: string; providerId?: string | null; commissionTiers?: Array<{ minRevenue: number; maxRevenue: number | null; rate: number }> })`:**
- `requireRole("admin")`
- Validates with Zod: availability must be "yes" | "no" | "unavailable", commission tiers must match schema
- Updates the `locationProducts` row by id
- Sets `updatedAt` to `new Date()`
- Returns `{ success: true }` or `{ error: string }`

**5. `addProduct(name: string)`** — per D-15:
- `requireRole("admin")`
- Inserts into `products` table (name must be unique — onConflictDoNothing)
- Gets the product ID (either newly created or existing)
- Fetches ALL location IDs from `locations` table
- Inserts a `locationProducts` row for EVERY location with `availability: "unavailable"` and `commissionTiers: []`
- Uses `onConflictDoNothing()` to handle cases where location_products rows already exist
- Returns `{ success: true, productId }` or `{ error: string }`

**File 2: `src/app/(app)/locations/[id]/products/page.tsx`**

Server component:
```tsx
import { LocationProductsClient } from "./location-products-client";
import { listLocationProducts, listAllProducts, listAllProviders } from "./actions";

export default async function LocationProductsPage({ params }: { params: Promise<{ id: string }> }) {
  const { id } = await params;
  const [locationProducts, allProducts, allProviders] = await Promise.all([
    listLocationProducts(id),
    listAllProducts(),
    listAllProviders(),
  ]);
  return <LocationProductsClient locationId={id} locationProducts={locationProducts} allProducts={allProducts} allProviders={allProviders} />;
}
```

"use client" component per D-16:

**Product configuration table:**
- Columns: Product Name | Availability | Provider | Commission Tiers | Actions
- Each row is one locationProduct record

**Availability column:**
- Select dropdown with options: "Yes", "No", "Unavailable"
- On change: calls `updateLocationProduct(id, { availability: value })`
- Visual indicator: green dot for "yes", grey for "no", red/muted for "unavailable"

**Provider column:**
- Select dropdown populated from `allProviders` list + "None" option
- On change: calls `updateLocationProduct(id, { providerId: value })`

**Commission Tiers column:**
- Shows current tiers as compact chips: e.g., "<3000: 5%" / ">3000: 12%"
- Click to expand inline editor (or small dialog):
  - Each tier: Min Revenue input, Max Revenue input (blank = no cap), Rate (%) input
  - "Add Tier" button to add another tier row
  - "Save" button calls `updateLocationProduct(id, { commissionTiers: [...] })`

**Add Product section:**
- Input field + "Add Product" button at the bottom of the table
- Calls `addProduct(name)` which creates the product globally and adds it to ALL locations
- After adding, refresh the list via `router.refresh()`

**Styling:**
