**Accent reserved for:**
1. "Import Records" primary CTA button background
2. "Run Preview" secondary CTA button text/border (outlined variant)
3. Progress bar fill during active import
4. Active/current step indicator in the import wizard stepper
5. Settings page card icon foreground (`text-wk-azure` on `bg-wk-sky-blue` icon square)

Accent is NOT used for table rows, log entries, badges (those use semantic colours), or general interactive hover states.

Source: `globals.css` token declarations + WeKnow brand guidelines (Azure is reserved for highlights and key elements) + `settings/page.tsx` established card pattern.

## UI State Machine

Phase 4's data-import page has six distinct states. Executor must implement all six; checker validates each.

| State | Trigger | Key UI Elements |
|-------|---------|-----------------|
| **S1: Initial** | Page load, no prior import | Empty state with "Connect Board" prompt; Step 1 active |
| **S2: Fetching columns** | Admin submits board ID | Skeleton loaders in mapping table; spinner on step label |
| **S3: Mapping review** | Board columns loaded | Editable mapping table (Monday.com column → Drizzle field); "Run Preview" CTA |
| **S4: Dry-run preview** | "Run Preview" clicked | Summary stats (total, mapped, warnings, duplicates); 10–20 row sample table; "Import Records" CTA; "Edit Mapping" back link |
| **S5: Import in progress** | "Import Records" confirmed in dialog | Progress bar (X / total); scrollable live log; "Cancel" disabled — import is not interruptible |
| **S6: Import complete** | Pagination loop exhausted | Success or partial-success summary; error count badge; full log available; "Run Again" link |

## Badge Semantics

| Label | Variant / Color | Meaning |
|-------|----------------|---------|
| Mapped | `default` (Azure bg) | Column auto-mapped to a Drizzle field |
| Warning | `secondary` (gold border) | Column mapped with caveats (type mismatch, optional field) |
| Unmapped | `outline` (mid-grey) | Column not mapped; will be stored in `notes` |
| Duplicate | `destructive` | Kiosk ID already exists in DB; will be skipped |
| Skipped | `outline` (mid-grey) | Row filtered out (missing required identifier column) |

Note: Use the existing `badge.tsx` variants — do not add custom badge classes.
