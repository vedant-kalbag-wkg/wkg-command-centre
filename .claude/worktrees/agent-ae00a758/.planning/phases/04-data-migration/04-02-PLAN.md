**3. Create `src/app/(app)/settings/data-import/data-import-client.tsx`:**

"use client" component implementing the 6-state machine from UI-SPEC:

**State management:** `useState` for:
- `step`: "connect" | "fetching" | "mapping" | "preview" | "importing" | "complete"
- `boardId`: string (input field, pre-filled from env)
- `mappings`: FieldMapping[] (from exploreBoardColumns)
- `columns`: BoardColumn[] (from exploreBoardColumns)
- `preview`: ImportPreview | null (from runDryImport)
- `sessionId`: string | null
- `progress`: ImportProgress | null
- `error`: string | null

**S1: Initial ("connect") state:**
- Board ID input field (pre-filled with `defaultBoardId` prop)
- "Connect Board" button (primary, Azure bg per UI-SPEC accent rules)
- Empty state text: "No imports yet" / "Connect your Monday.com board to preview and import kiosk records."

**S2: Fetching columns ("fetching") state:**
- Skeleton loaders in place of mapping table rows
- Spinner indicator on step label

**S3: Mapping review ("mapping") state:**
- Table with columns: Monday.com Column | Type | Status | Target Field
- Each row shows column title, type badge, status badge (Mapped/Unmapped per UI-SPEC badge semantics), and a Select dropdown for target field
- Select options: all kiosk fields + all location fields + "Skip (unmapped)"
- Auto-detected mappings pre-selected
- "Run Preview" CTA button (Azure outlined variant)

**S4: Dry-run preview ("preview") state — per D-06:**
- Summary stats bar: "{N} records found, {N} mapped, {N} warnings, {N} duplicates"
- If `preview.newStageNames.length > 0`: Alert showing "New pipeline stages will be created: {names}"
- If `preview.productNames.length > 0`: Section showing "Products found: {names}" and "Providers found: {names}"
- Sample records table (first 10-20 per D-06): Monday.com Name | Mapped Fields (key columns) | Warnings
- "Import Records" primary CTA (Azure bg)
- "Edit field mapping" back link
- Clicking "Import Records" opens a Dialog (destructive confirmation per UI-SPEC):
  - Heading: "Import {N} records?"
  - Body: "This will create new kiosk and location records in the database. Existing records with matching kiosk IDs will be skipped. This action cannot be undone."
  - "Import Records" + "Cancel" buttons

**S5: Import in progress ("importing") state — per D-07:**
- Progress bar component: value = `(progress.current / progress.total) * 100`
- Label: "Importing records... ({current} / {total})"
- ScrollArea (h-64) with live log feed — each entry: timestamp + level icon + message
  - Info: `text-sm font-mono text-wk-night-grey`
  - Warn: `text-sm font-mono text-wk-gold` (or `text-amber-600`)
  - Error: `text-sm font-mono text-wk-destructive`
- Auto-scroll to bottom on new log entries (useEffect + scrollIntoView on log container ref)
- Polling: `setInterval` calling `getImportProgress(sessionId)` every 1500ms, cleared on complete/unmount

**S6: Import complete ("complete") state:**
- Success: "Import complete. {N} kiosks and {N} locations created." (per UI-SPEC)
- With products: "Also imported: {N} products, {N} providers, {N} product configurations."
- Partial: "Import completed with {N} errors. {N} records imported successfully."
- Zero: "No records imported. Review warnings above."
- Error count badge (destructive variant) if errors > 0
- Full log still visible in ScrollArea
- "Run another import" link resets to S1

**Step indicator:** Three-step horizontal stepper at top of page:
- Step 1: "Connect Board" — active when step is connect/fetching
- Step 2: "Review Mapping" — active when step is mapping
- Step 3: "Preview & Import" — active when step is preview/importing/complete
- Active step: Azure text + border-bottom. Completed steps: checkmark icon. Future steps: grey text.
