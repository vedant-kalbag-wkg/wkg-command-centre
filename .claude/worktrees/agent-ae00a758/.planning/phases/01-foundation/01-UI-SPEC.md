## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge padding |
| sm | 8px | Compact element spacing, button icon gap, input inner padding |
| md | 16px | Default element spacing, form field gap, card padding |
| lg | 24px | Section padding, sidebar group gap |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Vertical page breathing room |
| 3xl | 64px | Login page vertical centering offset |

Exceptions:
- Touch targets: minimum 44×44px for all interactive controls (buttons, nav items, checkboxes) — per WCAG 2.5.5
- Sidebar width expanded: 248px fixed
- Sidebar width collapsed (icon-only): 56px fixed
- Login card max-width: 400px centered

## Typography

Font: Circular Pro. Bold weight always uses letter-spacing: -0.01em (brand kerning rule: -10 tracking).

| Role | Size | Weight | Line Height | Letter Spacing | Usage |
|------|------|--------|-------------|----------------|-------|
| Body | 14px | Book (400) | 1.5 | 0 | Form labels, table cells, descriptive copy, nav labels |
| Label | 12px | Book (400) | 1.4 | 0 | Helper text, badges, status indicators, timestamps |
| Heading | 20px | Bold (700) | 1.2 | -0.01em | Page titles, section headings, card titles |
| Display | 28px | Bold (700) | 1.15 | -0.01em | Login page headline only |

Constraints:
- Exactly 4 sizes, 2 weights (400 Book for body/label, 700 Bold for headings)
- No italic in UI — italic reserved for brand copywriting, not internal tool UI
- No uppercase body text — uppercase only acceptable for headings/subheadings per brand guidelines
- Minimum body text size in production: 14px (never 12px for body content)

## Interaction Contract

### Focus States
- All interactive elements: 2px solid `#00A6D3` focus ring, 2px offset
- No custom outline suppression without replacement — `outline: none` is banned without `ring` substitute
- Keyboard navigation must be fully functional across all Phase 1 pages

### Disabled States
- Disabled controls: 0.4 opacity on the control itself
- Disabled controls shown (not hidden) with Lucide `Lock` icon tooltip explaining why — per CONTEXT.md decision
- Tooltip text: "You don't have permission to [action]"
- Cursor: `not-allowed` on disabled interactive elements

### Loading States
- Threshold: show loading indicator after 300ms delay
- Spinner: Lucide `Loader2` with `animate-spin`, `#00A6D3` Azure colour, 20px size
- Skeleton: use shadcn `Skeleton` component for list/table content (grey pulse on `#F4F4F4`)
- Form submit: button enters loading state (spinner replaces icon, label unchanged, button disabled)
- Full-page loading: skeleton for app shell content area only — sidebar always visible

### Error States
- Inline form errors: below the field, `#F41E56` text, 12px Label size, Lucide `AlertCircle` 14px icon inline
- Toast notifications: Sonner (shadcn default) — top-right, 4s auto-dismiss
  - Error toast: red/destructive variant
  - Success toast: neutral/default variant (no green background — too noisy)
- Error page (session expired, 404): centered message with "Go to login" link, no chrome

### Transitions
- Sidebar collapse/expand: 150ms ease-out (width transition)
- Sidebar item hover: 100ms ease background-color
- Button hover: 100ms ease background-color — no transform, no scale
- Modal/dialog open: 150ms ease-out opacity + 8px translate-y enter
- Modal/dialog close: 100ms ease-in opacity exit
- Toast enter: 150ms ease-out slide-in from right
- No bounce, spring, or elastic easing in the app shell
- No `transform: scale()` on any hover state

### Form Validation
- Approach: react-hook-form + zod (per RESEARCH.md standard stack)
- Trigger: validate on blur, re-validate on change after first error
- Error display: inline below field — no summary panel at top of form
- Required field indicator: asterisk (*) in `#575A5C` Night Grey after label, not before
- Password field: show/hide toggle (Lucide `Eye`/`EyeOff`) on the right of input

## Page Contracts

### Login Page (`/login`)

**Layout:** Full-page centered, white background. WeKnow horizontal logo (mono-positive: Azure wink + Graphite text) top-center of card. Card: 400px max-width, 32px padding, `#FFFFFF` background, 1px `#ADADAD` border, 8px border-radius. No shadow beyond `0 2px 8px rgba(0,0,0,0.08)`.

**Elements:**
- Logo at top — horizontal variant, 120px width, centered
- Heading (20px Bold): "Sign in to WeKnow"
- Email input + label: "Email address"
- Password input + label: "Password" + show/hide toggle
- "Forgot password?" text link below password field, right-aligned, 14px Book, `#00A6D3` Azure
- Primary CTA button: "Sign in" — full width, Azure fill, white text, 44px height, 8px radius
- No "Sign up" link — invite-only (public signup blocked)

**Primary visual anchor:** Azure "Sign in" button — full-width, 44px height, draws eye after logo and form fields. Visual hierarchy flows: logo → heading → inputs → CTA.

**States:** loading (button spinner), error (inline below each field + toast for server errors), success (redirect to /kiosks).

### Password Reset Page (`/reset-password`)

**Layout:** Same centered card layout as login. Back link top-left of card.

**Elements:**
- Back link: "← Back to sign in" — 14px, `#575A5C`, no underline, `#00A6D3` on hover
- Heading (20px Bold): "Reset your password"
- Body copy (14px): "Enter your email and we'll send you a link to reset your password."
- Email input + label: "Email address"
- Primary CTA: "Send reset link" — full width, Azure fill

**States:** idle, loading, success (swap form for confirmation message), error (inline + toast).

**Success state message:**
- Heading (20px Bold): "Check your inbox"
- Body: "We sent a reset link to [email]. Check your spam folder if it doesn't arrive within a few minutes."
- Link: "Back to sign in"

### Set Password Page (`/set-password`)

Used for both initial invite acceptance and password reset confirmation.

**Elements:**
- Heading: "Set your password" (first login) / "Choose a new password" (reset)
- New password input + show/hide toggle
- Confirm password input + show/hide toggle
- Primary CTA: "Set password" / "Update password"
- Password requirements: inline below new password field, 12px Label, `#575A5C` — "At least 8 characters"

### App Shell

**Layout:**
- Sidebar: 248px fixed left, `#121212` Graphite background, full viewport height
- Content area: fills remaining width, `#FFFFFF` white background, 24px padding
- Sidebar collapsed (icon-only): 56px width — icons centered, labels hidden, tooltips on hover
- Mobile (< 768px): sidebar hidden, hamburger trigger in top-left of content header

**Sidebar structure (top to bottom):**
1. Logo area: WeKnow logo (mono-negative: white) — 16px horizontal padding, 24px top/bottom padding
2. Separator
3. Navigation items: Kiosks (LayoutGrid icon), Locations (MapPin icon), Settings (Settings icon)
4. Separator
5. Footer: user avatar + name + role badge + logout button (flush bottom)

**Nav item anatomy:**
- 40px height, 8px horizontal padding, 8px border-radius
- Default: white text at 0.7 opacity, no background
- Hover: white text at 1.0 opacity, `rgba(0,166,211,0.20)` background
- Active: white text at 1.0 opacity, `rgba(0,166,211,0.20)` background, 2px `#00A6D3` Azure left border
- Icons: 18px Lucide icons, 8px gap to label

**Sidebar collapse trigger:** SidebarTrigger (shadcn) — ChevronLeft/ChevronRight icon, positioned at bottom of logo area. 150ms ease-out transition on width.

**Content header (inside content area):**
- 56px height, bottom border `1px solid #ADADAD`
- Page title (20px Bold) left-aligned
- No eyebrows, no subtitle copy, no gradient text

### Settings > Users Page (`/settings/users`)

**Layout:** Standard content area. Table fills width.

**Elements:**
- Page title: "Users"
- "Invite user" button (top-right): Primary CTA, Azure fill, 40px height, Lucide `UserPlus` icon + "Invite user" label
- Table columns: Name, Email, Role (Badge), Status, Joined, Actions
- Role badges: Admin (`#121212` bg, white text), Member (`#F4F4F4` bg, `#121212` text), Viewer (`#E5F1F9` bg, `#575A5C` text) — all 6px radius, 6px/12px padding
- Actions column: dropdown menu with "Change role" and "Deactivate" — destructive item (`#F41E56` text)
- Deactivated users: row at 0.5 opacity, status badge "Inactive"

**Primary visual anchor:** "Invite user" button (top-right, Azure fill) — draws attention as the primary action on the page. Visual hierarchy flows: page title → invite button → table data.

**Invite user dialog:**
- Modal (shadcn Dialog), 480px max-width
- Title: "Invite user"
- Email input + label
- Role select: dropdown (Admin / Member / Viewer)
- CTA: "Send invite" — Azure fill
- Cancel link below CTA

## Accessibility Baseline

- Colour contrast: all text must meet WCAG AA 4.5:1 minimum
  - `#121212` on `#FFFFFF`: 18.1:1 — passes
  - `#FFFFFF` on `#00A6D3` (Azure buttons): 3.1:1 — FAILS AA for normal text; use `#FFFFFF` text only on buttons ≥ 20px Bold (large text threshold 3:1) or darken button hover to `#008BB2` Sea Blue for sufficient contrast
  - **Action required in Plan 01-01:** Primary button hover state = `#008BB2` Sea Blue (not Azure 80% tint) to ensure contrast on white text
  - `#575A5C` on `#FFFFFF`: 5.74:1 — passes
  - `#ADADAD` on `#FFFFFF`: 2.85:1 — FAILS; never use Mid Grey for body text, only for borders and decorative elements
- All form inputs: associated `<label>` via `htmlFor`/`id`
- Error messages: `role="alert"` or `aria-live="polite"` on error containers
- Dialog: focus trap via shadcn Dialog (Radix), focus returns to trigger on close
- Sidebar: `aria-label="Main navigation"` on `<nav>`, active item has `aria-current="page"`
- Images: login logo has `alt="WeKnow"` — no decorative images in Phase 1
