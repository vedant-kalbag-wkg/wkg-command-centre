Create `src/app/(app)/locations/actions.ts` with all location server actions. Follow the exact pattern from kiosks/actions.ts (try/catch, requireRole, Zod v4 via `import { z } from "zod/v4"`, return `{ success }` or `{ error }`). Every mutation calls `writeAuditLog()`.

Actions to implement:

1. `createLocation(data)` — Zod schema validates: name (required, 1-200 chars), address (optional), latitude/longitude (optional numbers), starRating (optional 1-5 integer), roomCount (optional positive integer), hotelGroup (optional), sourcedBy (optional), notes (optional), contractValue (optional), contractStartDate (optional), contractEndDate (optional), contractTerms (optional). `requireRole("admin", "member")`. Insert into locations. Write audit log action "create". Return `{ success: true, id }`.

2. `getLocation(id)` — Fetch location by ID. Also fetch all kioskAssignments for this location (JOIN kiosks for kiosk details). Apply `redactSensitiveFields()` based on current user role. Return location + assigned kiosks (current and historical).

3. `updateLocationField(locationId, field, value, oldValue)` — `requireRole("admin", "member")`. For banking fields, additionally check `canAccessSensitiveFields(role)`. Updates single field. Writes audit log. Returns `{ success: true }`.

4. `archiveLocation(locationId)` — `requireRole("admin", "member")`. Sets archivedAt. Writes audit log action "archive". Returns `{ success: true }`.

5. `listLocations()` — All locations where archivedAt IS NULL. Returns array with basic fields (id, name, address, hotelGroup, starRating, roomCount, kiosk count).

6. `getContractUploadUrl(fileName, contentType)` — `requireRole("admin", "member")`. Creates S3 presigned PUT URL using `@aws-sdk/client-s3` + `@aws-sdk/s3-request-presigner`. S3 key pattern: `contracts/${crypto.randomUUID()}-${fileName}`. Expires in 300 seconds. Returns `{ presignedUrl, s3Key }`. If AWS env vars are not configured, return `{ error: "File upload not configured. Contact your administrator." }`.

7. `saveContractDocument(locationId, s3Key, fileName)` — `requireRole("admin", "member")`. Fetches current contractDocuments array, appends new entry `{ fileName, s3Key, uploadedAt: new Date().toISOString() }`. Updates location. Writes audit log for field "contractDocuments". Returns `{ success: true }`.

8. `removeContractDocument(locationId, s3Key)` — `requireRole("admin", "member")`. Filters out the document with matching s3Key from contractDocuments array. Updates location. Writes audit log. Returns `{ success: true }`.

9. `updateKeyContacts(locationId, contacts)` — `requireRole("admin", "member")`. Validates contacts array with Zod (each entry: name required, role optional, email optional z.email(), phone optional). Updates keyContacts JSONB field. Writes audit log. Returns `{ success: true }`.

10. `updateBankingDetails(locationId, bankingDetails)` — `requireRole("admin")`. Only admin can modify banking. Updates bankingDetails JSONB. Writes audit log (oldValue/newValue redacted to "[REDACTED]" in audit log for security). Returns `{ success: true }`.
