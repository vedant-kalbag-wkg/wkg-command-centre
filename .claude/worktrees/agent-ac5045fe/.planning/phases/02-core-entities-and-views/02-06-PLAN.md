<task type="auto">
  <name>Task 3: Fix key contacts editor to save on blur instead of each keystroke</name>
  <files>src/components/locations/key-contacts-editor.tsx</files>
  <action>
    Change the save behavior so field changes only update local state (no server call on each keystroke). Persist to server only on input blur.

    1. In `updateField` (line 56-62): Remove the `persist(updated)` call. Only call `setContacts(updated)` — this updates local state without triggering a server round-trip.

    2. Add an `onBlur` handler to each Input that calls `persist(contacts)` with current contacts state. Use a ref to get the latest contacts value (since blur handler closures may be stale):
       ```tsx
       const contactsRef = useRef(contacts);
       contactsRef.current = contacts;

       const handleBlur = useCallback(() => {
         persist(contactsRef.current);
       }, [persist]);
       ```

    3. Add `onBlur={handleBlur}` to all four Input elements (name, role, email, phone).

    4. Remove `isSaving` from the Input `disabled` prop — saving should happen in the background without locking the form. Change `disabled={disabled || isSaving}` to just `disabled={disabled}`.

    5. Keep the `persist` call in `removeContact` — removing a row should save immediately since the user explicitly clicked the remove button.

    6. Keep the debounce in `persist` (useful for rapid blur events like Tab through fields) but increase to 1000ms since blur is less frequent than keystrokes.
  </action>
