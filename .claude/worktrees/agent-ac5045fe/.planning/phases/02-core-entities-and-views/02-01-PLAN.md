**Step 1 — Create `src/app/(app)/kiosks/actions.ts`:**
All kiosk server actions following the established pattern (try/catch, requireRole, Zod v4 validation, return `{ success }` or `{ error }`). Every mutation calls `writeAuditLog()`.

Actions to implement:
1. `createKiosk(data)` — Zod schema validates all kiosk fields. `requireRole("admin", "member")`. Inserts into kiosks table. If `pipelineStageId` is not provided, assigns the default stage (query pipelineStages where isDefault=true). Writes audit log with action "create". Returns `{ success: true, id: newKiosk.id }`.

2. `getKiosk(id)` — Fetches kiosk by ID, joins pipelineStages for stage name/color, fetches current assignment (kioskAssignments where unassignedAt IS NULL), fetches assignment history (all kioskAssignments ordered by assignedAt DESC), joins location name for each assignment. Returns full kiosk object with stage and assignments.

3. `updateKioskField(kioskId, field, value, oldValue)` — `requireRole("admin", "member")`. Updates single field on kiosks table. Writes audit log with field name, old value, new value. Returns `{ success: true }`.

4. `archiveKiosk(kioskId)` — `requireRole("admin", "member")`. Sets `archivedAt` to `new Date()`. Writes audit log with action "archive". Returns `{ success: true }`.

5. `assignKiosk(kioskId, locationId, reason?)` — `requireRole("admin", "member")`. First close any current assignment (set `unassignedAt` on existing row where `unassignedAt IS NULL`). Then insert new kioskAssignment row. Writes audit log with action "assign", field "venue", newValue = location name. Returns `{ success: true }`.

6. `reassignKiosk(kioskId, newLocationId, reason?)` — Same as assignKiosk but explicitly closes previous assignment and writes audit log with action "assign", oldValue = old location name, newValue = new location name.

7. `listKiosks()` — Fetches all kiosks where `archivedAt IS NULL`, joins pipelineStages for stage name/color, joins current assignment for venue name. Returns array. Used by list page.

8. `listPipelineStages()` — Fetches all pipeline stages ordered by position ASC. Used by forms and kanban.

9. `listLocationsForSelect()` — Fetches all locations (id, name) where archivedAt IS NULL for venue assignment dropdown.
