5. Create `drizzle.config.ts` at project root — exactly as RESEARCH.md code example:
   ```typescript
   import { config } from "dotenv";
   import { defineConfig } from "drizzle-kit";

   export default defineConfig({
     schema: "./src/db/schema.ts",

   });
   ```

7. Create `src/db/schema.ts` with the COMPLETE database schema for all downstream phases. This includes:

   **Better Auth tables** (run `npx auth@latest generate` after auth setup in Plan 01-02, but define placeholders now):
   - `user` table: id, name, email, emailVerified, image, createdAt, updatedAt, role (text, default "member"), banned (boolean), banReason (text), banExpires (timestamp)
   - `session` table: id, expiresAt, token, createdAt, updatedAt, ipAddress, userAgent, userId (FK to user)
   - `account` table: id, accountId, providerId, userId, accessToken, refreshToken, idToken, accessTokenExpiresAt, refreshTokenExpiresAt, scope, password, createdAt, updatedAt
   - `verification` table: id, identifier, value, expiresAt, createdAt, updatedAt

   **Application tables:**
   - `kiosks` table: id (uuid PK), kioskId (text unique), outletCode (text), customerCode1 (text), customerCode2 (text), hardwareModel (text), hardwareSerialNumber (text), softwareVersion (text), cmsConfigStatus (text), installationDate (timestamp), deploymentPhaseTags (text array or jsonb), maintenanceFee (numeric), freeTrialStatus (boolean default false), freeTrialEndDate (timestamp), regionGroup (text), pipelineStageId (uuid FK to pipeline_stages), notes (text), createdAt (timestamp with tz), updatedAt (timestamp with tz)
   - `locations` table: id (uuid PK), name (text), address (text), latitude (doublePrecision), longitude (doublePrecision), starRating (integer), roomCount (integer), keyContacts (jsonb — array of {name, role, email, phone}), hotelGroup (text), sourcedBy (text), bankingDetails (jsonb — sensitive), contractValue (numeric — sensitive), contractStartDate (timestamp), contractEndDate (timestamp), contractTerms (text — sensitive), contractDocuments (jsonb — array of {fileName, s3Key, uploadedAt}), notes (text), createdAt (timestamp with tz), updatedAt (timestamp with tz)
   - `kioskAssignments` table: id (uuid PK), kioskId (uuid FK), locationId (uuid FK), assignedAt (timestamp with tz), unassignedAt (timestamp with tz nullable — null = current), reason (text), assignedBy (uuid FK to user), assignedByName (text — denormalized), createdAt (timestamp with tz)
   - `pipelineStages` table: id (uuid PK), name (text), position (doublePrecision — FLOAT8), color (text), isDefault (boolean default false), createdAt (timestamp with tz), updatedAt (timestamp with tz)
   - `auditLogs` table: id (uuid PK), actorId (uuid), actorName (text — denormalized), entityType (text — "kiosk" | "location" | "user"), entityId (uuid), entityName (text — denormalized), action (text — "create" | "update" | "delete" | "assign" | "unassign"), field (text nullable), oldValue (text nullable), newValue (text nullable), metadata (jsonb nullable), createdAt (timestamp with tz)
   - `userViews` table: id (uuid PK), userId (uuid FK to user), name (text), entityType (text — "kiosk" | "location"), config (jsonb — {filters, sort, groupBy, columns, etc.}), isDefault (boolean default false), createdAt (timestamp with tz), updatedAt (timestamp with tz)

   Seed default pipeline stages after schema push:
   Prospect (pos 1.0), On Hold (2.0), Delivered to Region (3.0), Awaiting Configuration (4.0), Configured (5.0), Ready to Launch (6.0), Live (7.0), Offline (8.0), Decommissioned (9.0).
