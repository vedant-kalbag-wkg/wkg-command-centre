## Spacing Scale

Declared values (multiples of 4 only — 8-point scale):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding, milestone marker padding |
| sm | 8px | Compact element spacing, toolbar item gaps, filter chip gaps |
| md | 16px | Default panel/card padding, toolbar height padding, form field spacing |
| lg | 24px | Section padding, Gantt left-grid column padding, calendar cell padding |
| xl | 32px | Layout gaps between toolbar and main view area |
| 2xl | 48px | Major section breaks, empty state top padding |
| 3xl | 64px | Page-level spacing (not commonly needed within Gantt/Calendar view area) |

**Exceptions:**
- Gantt row height: 44px minimum — touch-target accessibility for drag handles
- Calendar event minimum touch target height: 24px (enforced by react-big-calendar default; do not reduce below this)
- Gantt diamond milestone marker: 12px × 12px (rotated square, not grid-scale — purely visual)

## Typography

| Role | Size | Weight | Line Height | Tracking | Usage |
|------|------|--------|-------------|---------|-------|
| Body | 14px | 400 (Book) | 1.5 | 0 | Gantt row labels, calendar event body, filter labels, table cells |
| Label | 12px | 400 (Book) | 1.4 | 0 | Badges, milestone type tags, toolbar button labels, column headers |
| Heading | 16px | 600 (Bold) | 1.25 | -0.01em | Installation name in Gantt rows, section group headers, card titles |
| Display | 20px | 600 (Bold) | 1.2 | -0.01em | Page title ("Installations"), AppShell title only |

**Rules:**
- Exactly 2 weights in use: 400 (Book) and 600 (Bold). No intermediate weights.
- Bold weights (600) always use -0.01em tracking (WeKnow brand: -10 kerning)
- All other weights use tracking 0
- Never use body text in uppercase; label text may use uppercase for column headers only
- Font family: `var(--font-sans)` — resolves to Circular Pro stack from globals.css

## Color

| Role | Value | Tailwind Token | Usage |
|------|-------|----------------|-------|
| Dominant (60%) | #FFFFFF | `bg-background` / `bg-wk-white` | Page background, Gantt timeline area, Calendar grid background, card surfaces |
| Secondary (30%) | #F4F4F4 | `bg-wk-light-grey` / `bg-secondary` | Gantt left-grid panel background, toolbar background, sidebar (already #121212 — existing), group header rows |
| Accent (10%) | #00A6D3 | `bg-wk-azure` / `text-primary` | See accent reserved-for list below |
| Destructive | #F41E56 | `bg-wk-destructive` / `text-destructive` | Delete Installation button, Delete Milestone button — confirmation dialogs only |

**Accent (#00A6D3) reserved for:**
1. Active tab indicator underline (Table / Kanban / Gantt / Calendar tab strip)
2. "Add Installation" primary CTA button background
3. Gantt bar fill colour for the current user's assigned installation (when region colour is not set)
4. Milestone quick-add popover confirm button
5. Focus ring on all interactive elements (`--ring: var(--wk-azure)` — already set in globals.css)
6. Hover state background for sidebar nav item (existing pattern: `rgba(0,166,211,0.15)`)
7. Pending-change indicator border on a modified Gantt bar

**Region colour palette (for Gantt bars and Calendar event blocks — color-coded by region):**

Colours must be drawn from the WeKnow secondary palette only — one colour per region, consistently applied across Gantt and Calendar. Suggested mapping (executor resolves against actual region data):

| Region slot | Colour | Hex |
|-------------|--------|-----|
| Region A | Azure | #00A6D3 |
| Region B | Sea Blue | #008BB2 |
| Region C | Night Blue | #0C2752 |
| Region D | Night Grey | #575A5C |
| Region E | Gold | #F4BA1E |
| Region F+ | Mid Grey | #ADADAD |

Region colours applied at 80% opacity on Calendar event blocks (to keep text legible on white background). On Gantt bars, use solid colour at full opacity.

**Pending-change state (Gantt drag unsaved):**
- Modified bar: region colour at 40% opacity (lighter shade) + 2px dashed `--wk-azure` border
- "Apply" button: Azure background, adjacent right of the modified bar

**Secondary semantic colours:**
- Success: #68D871 (`--wk-success`) — used only for "Installation complete" status badge
- Warning: #F4BA1E (`--wk-gold`) — used only for trial expiry events on Calendar (dot markers) when expiry is within 30 days

## Focal Point

**Primary visual anchor (Gantt screen):** The Gantt timeline bar area is the dominant visual element — installation bars spanning the date grid draw the eye first. The left-side data grid (installation names, team column) is supporting context; it is visually quieter (lighter background, smaller type) so as not to compete with the bar area.

**Primary visual anchor (Calendar screen):** Multi-day installation blocks are the dominant element — they span multiple date cells and carry region colour. Milestone diamond markers and trial expiry dots are secondary signals layered on top.

## Interaction Contracts

### Gantt Interactions

| Interaction | Trigger | Visual Feedback | Outcome |
|-------------|---------|-----------------|---------|
| Drag bar end (resize) | Mouse/touch drag on bar edge | Bar width changes; bar shades to region-colour@40% + dashed azure border | Date range shown in tooltip; "Apply" button appears |
| Drag whole bar (shift) | Mouse/touch drag on bar body | Bar translates; pending-change visual as above | Date range shifts; "Apply" button appears |
| Apply pending change | Click "Apply changes" button | Button shows spinner; bar returns to solid region colour | Server action saves new dates; toast "Timeline updated" |
| Discard pending change | Click "Discard changes" link (secondary, adjacent to Apply) | Bar snaps back to original dates | No server call; pending state cleared |
| Click on date position in row | Click on empty area of a row | MilestoneQuickAddPopover opens at click position | Popover pre-fills target date from click position |
| Milestone quick-add submit | Enter name + select type → click "Add milestone" | Popover closes; diamond marker appears on bar | Server action creates milestone; toast "Milestone added" |
| Group header collapse | Click chevron on group header row | Chevron rotates; rows animate out | Rows hidden; group count badge shows N kiosks |
| Switch grouping | Select from grouping dropdown | Gantt re-renders with new group rows | Grouped by selected field (region or deployment phase) |
| Zoom controls | Click Day / Week / Month buttons | Timeline axis rescales | Default zoom: Month |

### Calendar Interactions

| Interaction | Trigger | Visual Feedback | Outcome |
|-------------|---------|-----------------|---------|
| Click installation block | Click multi-day event block | CalendarEventPopover opens | Shows: installation name, dates, region, status + "View installation" link |
| Click milestone marker | Click diamond pin | CalendarEventPopover opens | Shows: milestone name, type, date, linked installation + "View installation" link |
| Click trial expiry dot | Click dot marker | CalendarEventPopover opens | Shows: kiosk name, trial end date + "View kiosk" link |
| Switch view (month/week/day) | Click view toggle buttons | Calendar re-renders with new density | Default view: Month |
| Filter change | Select from filter dropdowns | Calendar events re-filter immediately (client-side) | No page reload |
| Navigate months/weeks | Click prev/next arrows | Calendar navigates; skeleton shows during data fetch if new month | Events load for new period |

### Saved Views — Gantt and Calendar

Extends existing View Engine. The saved view stores: active grouping (Gantt), active zoom (Gantt), active view mode (Calendar: month/week/day), active filter selections, and view_type field (`gantt` or `calendar`). Behaviour identical to Phase 2 SavedViewsBar pattern.

### URL Query Parameter

`?view=table|kanban|gantt|calendar` — controls which tab is active on the Kiosks page. Existing table and kanban tabs must also adopt this parameter. Default: `?view=table`.

## Accessibility

- All Gantt drag operations must have keyboard alternatives: focus bar → press Space to enter "move mode" → arrow keys adjust dates → Enter to confirm → Escape to cancel
- Milestone diamond markers: `role="img"` with `aria-label="{name} milestone — {date}"`
- Calendar event blocks: `role="button"` with `aria-label="{type}: {name}, {date range}"`
- Filter dropdowns use shadcn Select — keyboard navigable by default
- Pending-change state must not rely on colour alone — use dashed border in addition to opacity change
- Grouping collapse: `aria-expanded` on the trigger button
- Minimum contrast ratio 4.5:1 for all text on coloured backgrounds — test region colours with white text
