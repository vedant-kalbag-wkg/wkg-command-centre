   - `id`: uuid PK defaultRandom
   - `name`: text notNull

   - `status`: text notNull default "planned" (values: "planned" | "active" | "complete")
   - `plannedStart`: timestamp("planned_start", { withTimezone: true }) nullable
   - `plannedEnd`: timestamp("planned_end", { withTimezone: true }) nullable
   - `createdAt`: timestamp("created_at", { withTimezone: true }).defaultNow().notNull()
   - `updatedAt`: timestamp("updated_at", { withTimezone: true }).defaultNow().notNull()

2. **milestones** table:
   - `id`: uuid PK defaultRandom
   - `installationId`: uuid("installation_id").notNull().references(() => installations.id, { onDelete: "cascade" })
   - `name`: text notNull
   - `type`: text notNull (values: "contract_signing" | "go_live" | "review_date" | "other")
   - `targetDate`: timestamp("target_date", { withTimezone: true }).notNull()
   - `createdAt`: timestamp("created_at", { withTimezone: true }).defaultNow().notNull()

3. **installationKiosks** table (join table):
   - `installationId`: uuid("installation_id").notNull().references(() => installations.id, { onDelete: "cascade" })
   - `kioskId`: uuid("kiosk_id").notNull().references(() => kiosks.id, { onDelete: "cascade" })
   - No separate PK — use composite primary key or just the two FK columns

4. **installationMembers** table (join table):

**A. Create `src/app/(app)/installations/actions.ts`** with these server actions following the existing kiosks/actions.ts pattern:

1. `createInstallation(data)` — Zod validated: { name: string (required, min 1), region?: string, status?: "planned"|"active"|"complete" (default "planned"), plannedStart?: string (ISO date), plannedEnd?: string (ISO date) }. Inserts into installations table. Writes audit log (entityType: "installation", action: "create"). Returns `{ success: true, id: string }` or `{ error: string }`.

2. `listInstallations()` — Returns all installations with their milestones and members joined. Use `requireRole("member")` guard (any authenticated user). Query joins: LEFT JOIN milestones, LEFT JOIN installationMembers + user table for member names. Return shape: `{ ...installation, milestones: Milestone[], members: { userId: string, userName: string, role: string }[] }`.

3. `getInstallation(id: string)` — Single installation with milestones, members (with user names), and linked kiosks. Returns full detail or `{ error: "Not found" }`.

4. `updateInstallation(id, data)` — Zod validated same fields as create (all optional). Updates installations table. Writes audit log for each changed field. Returns `{ success: true }` or `{ error: string }`.

5. `deleteInstallation(id: string)` — Hard delete (cascade deletes milestones, members, kiosk links). Writes audit log (action: "delete"). Returns `{ success: true }` or `{ error: string }`.

6. `createMilestone(data)` — Zod validated: { installationId: string, name: string, type: "contract_signing"|"go_live"|"review_date"|"other", targetDate: string (ISO date) }. Inserts into milestones table. Returns `{ success: true, id: string }` or `{ error: string }`.

7. `deleteMilestone(id: string)` — Hard delete. Returns `{ success: true }` or `{ error: string }`.

8. `addInstallationMember(data)` — Zod validated: { installationId: string, userId: string, role: "project_lead"|"installer"|"coordinator" }. Inserts into installationMembers. Returns `{ success: true }` or `{ error: string }`.

9. `removeInstallationMember(installationId: string, userId: string)` — Deletes from installationMembers. Returns `{ success: true }` or `{ error: string }`.

10. `linkKioskToInstallation(installationId: string, kioskId: string)` — Inserts into installationKiosks. Returns `{ success: true }` or `{ error: string }`.

11. `unlinkKioskFromInstallation(installationId: string, kioskId: string)` — Deletes from installationKiosks. Returns `{ success: true }` or `{ error: string }`.

All actions use `"use server"` directive, `requireRole("member")` guard (any auth user can manage installations), try/catch with Error union returns, and Zod validation.
