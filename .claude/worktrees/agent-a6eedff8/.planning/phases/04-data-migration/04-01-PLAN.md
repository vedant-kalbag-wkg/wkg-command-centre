Create `src/lib/monday-client.ts` with the following exports:

**1. Core GraphQL client (`mondayQuery<T>`):**
- POST to `https://api.monday.com/v2`
- Headers: `Authorization: ${process.env.MONDAY_API_TOKEN}`, `Content-Type: application/json`, `API-Version: 2024-01`
- Throws `Error("MONDAY_API_TOKEN not set")` if env var missing
- Throws on HTTP errors or GraphQL `data.errors`
- Returns `data.data as T`

**2. Retry wrapper (`mondayQueryWithRetry<T>`):**
- Wraps `mondayQuery` with exponential backoff per D-10
- Initial delay 1000ms, doubles each retry, cap at 32000ms
- Max 5 retries
- Only retries on rate-limit errors (message includes "rate" or HTTP 429)
- Accepts optional `onRetry?: (attempt: number, delayMs: number) => void` callback for logging
- Non-rate-limit errors thrown immediately

**3. Board columns fetcher (`fetchBoardColumns(boardId: string)`):**
Returns `Array<{ id: string; title: string; type: string }>` — the column metadata for a board.

GraphQL query:
```graphql
query ($boardId: [ID!]) {
  boards(ids: $boardId) {
    id
    name
    columns { id title type }
  }
}
```

**4. Paginated items fetcher (`fetchAllItems(boardId: string, onProgress?)`):**
Async generator that yields `MondayItem[]` pages.

Initial query fetches `items_page(limit: 500)` with:
```graphql
items {
  id
  name
  group { id title }
  column_values {
    id type text value
    ... on StatusValue { label }
    ... on DateValue { date }
    ... on MirrorValue { display_value }
  }
  subitems {
    id
    name
    column_values { id type text value }
  }
}
```

Continuation via `next_items_page(cursor: $cursor, limit: 500)` until cursor is null.
Uses `mondayQueryWithRetry` for each page call.
Calls `onProgress?.({ page, itemCount })` after each page.

NOTE per D-17: subitems are nested directly in the items query — NOT a separate API call. The `subitems { ... }` field on each item returns the subitem data from subboard 1413119149.

**2. `autoDetectMappings(columns: BoardColumn[])`** — per D-02:
- For each Monday.com column, check exact match in KNOWN_FIELD_MAP
- If no exact match, try case-insensitive includes matching
- Returns `Array<{ mondayColumnId: string; mondayTitle: string; mondayType: string; targetTable: "kiosk" | "location" | null; targetField: string | null; status: "mapped" | "unmapped" }>`

**3. `mapMondayItemToKiosk(item: MondayItem, mappings: FieldMapping[], existingStages: Map<string, string>)`**:
- Extracts kiosk fields from item.column_values using the confirmed mappings
- For Status column: uses `label` field (StatusValue fragment) — if no matching pipeline stage, returns `{ newStageName }` for auto-creation per D-04
- Item `name` is used as `kioskId` (the Monday.com row name is the kiosk identifier)
- Returns `{ kioskData: Partial<typeof kiosks.$inferInsert>; unmappedValues: Record<string, string>; newStageName?: string }`
- Unmapped column values formatted as "ColumnTitle: value" for D-09 notes field

**4. `mapMondayItemToLocation(item: MondayItem, mappings: FieldMapping[])`**:
- Extracts location fields from item.column_values
- Item `name` used as location name
- Returns `Partial<typeof locations.$inferInsert>`

**5. `extractLocationName(item: MondayItem)`** — returns the hotel/location name for deduplication (per D-03)

**6. `mapSubitemsToLocationProducts(subitems: MondaySubitem[])`** — per D-13, D-17:
- For each subitem:
  - `name` → product name (e.g., "Transfers", "Airport Shuttle")
  - Column `label2__1` → provider name (e.g., "Uber", "WeKnow")
  - Column `color5__1` → availability ("Yes"/"No"/"Unavailable" → lowercase)
  - Column `dup__of_commission9__1` → commission rate for <£3000pm revenue
  - Column `numeric_mkse455j` → commission rate for >£3000pm revenue
- Returns `Array<{ productName: string; providerName: string | null; availability: "yes" | "no" | "unavailable"; commissionTiers: Array<{ minRevenue: number; maxRevenue: number | null; rate: number }> }>`
- Commission tiers built as: `[{ minRevenue: 0, maxRevenue: 3000, rate: <£3000 value }, { minRevenue: 3000, maxRevenue: null, rate: >£3000 value }]`
- If only one commission value exists, create single tier with `maxRevenue: null`

**File 2: `src/app/(app)/settings/data-import/actions.ts`**

Create server actions following the existing pattern from `settings/users/actions.ts`:

**1. `exploreBoardColumns(boardId: string)`** — per MIGR-01:
- `"use server"`, `requireRole("admin")`
- Validates boardId with Zod (non-empty string)
- Calls `fetchBoardColumns(boardId)` from monday-client
- Calls `autoDetectMappings(columns)` from field-mapper
- Returns `{ success: true, columns, mappings }` or `{ error: string }`

**2. `runDryImport(boardId: string, mappings: FieldMapping[])` — per MIGR-02:
- `"use server"`, `requireRole("admin")`
- Fetches ALL items via `fetchAllItems(boardId)` (collects all pages into array)
- For each item: run `mapMondayItemToKiosk` and `mapMondayItemToLocation`
- Check duplicates against existing kiosks in DB (query by kioskId)
- Parse subitems via `mapSubitemsToLocationProducts`
- Collect unique product names and provider names across all subitems
- Returns `ImportPreview` with first 20 records as sample, counts, warnings, product/provider lists
- Does NOT write to database

**3. `runFullImport(boardId: string, mappings: FieldMapping[])`** — per MIGR-01, MIGR-03:
- `"use server"`, `requireRole("admin")`
- Generates a `sessionId` (crypto.randomUUID)
- Stores initial progress in module-level `Map<string, ImportProgress>` (importSessions)
- Kicks off the import in a detached async (don't await — let client poll):
  1. Fetch all items (already done by dry-run, but re-fetch for freshness)
  2. Deduplicate locations by hotel name (per D-03) — insert unique locations, collect ID map
  3. Auto-create pipeline stages for unknown status values (per D-04)
  4. Insert/upsert products from distinct subitem product names (per D-14) — `onConflictDoNothing()`
  5. Insert/upsert providers from distinct subitem provider names (per D-14) — `onConflictDoNothing()`
  6. For each item: insert kiosk (per D-08: `onConflictDoNothing()` on kioskId), create kiosk_assignment, create location_products from subitems
  7. Update progress after each item
  8. Unmapped column values → concatenate into kiosk/location `notes` field (per D-09)
  9. On error per item: log error, continue (per D-11), increment error count
  10. On completion: set status to "complete" with result summary
- Returns `{ success: true, sessionId }` immediately

**4. `getImportProgress(sessionId: string)`**:
- `"use server"`, `requireRole("admin")`
- Reads from module-level `importSessions` Map
- Returns the `ImportProgress` object or `{ error: "Session not found" }`

**Module-level state:**
```typescript
const importSessions = new Map<string, ImportProgress>();
```
This is acceptable for a one-time admin tool on a single Next.js server instance (per RESEARCH.md Pattern 5).
