**Step 1 — Create `src/app/(app)/settings/pipeline-stages/actions.ts`:**
Server actions for pipeline stage management. `requireRole("admin")` on all mutations.

Actions:
1. `createStage(name, color)` — Find the max position in pipelineStages, set new position = maxPosition + 1000. Insert stage. Return `{ success: true, id }`.

2. `updateStage(stageId, data: { name?, color?, isDefault? })` — Update stage fields. If setting isDefault=true, first set all other stages to isDefault=false. Return `{ success: true }`.

3. `deleteStage(stageId, reassignToStageId)` — First count kiosks with this pipelineStageId. If count > 0 and no reassignToStageId provided, return `{ error: "Stage has kiosks", kioskCount: N }`. If reassignToStageId provided, update all kiosks to the new stage. Then delete the stage. Cannot delete the last remaining stage — return error. Return `{ success: true }`.

4. `reorderStage(stageId, afterPosition, beforePosition)` — Calculate new position as `(afterPosition + beforePosition) / 2`. If moving to first position: `newPosition = beforePosition / 2`. If moving to last position: `newPosition = afterPosition + 1000`. Update stage position. Return `{ success: true }`.

5. `getStageKioskCount(stageId)` — Count kiosks with this pipelineStageId. Return `{ count: N }`.

- DndContext with closestCenter collision detection
- Each column is a droppable with id = stageId
- Cards within each column in a SortableContext (but we're doing cross-column drops, not intra-column sorting — use `verticalListSortingStrategy` for drop detection)
- onDragEnd: if `over` exists and `active.id !== over.id` (different column), apply optimistic update (move kiosk to new stage in local state), then call `updateKioskField(kioskId, "pipelineStageId", newStageId, oldStageId)`. - Column overflow: use ScrollArea for vertical scroll within each column
- Empty column: "No kiosks in this stage" in 12px muted text, centered

- Column header: field value + count badge (no colored dot)
- Drag is DISABLED — SortableContext has `disabled={true}`
- Info banner below header bar: "Switch to stage grouping to drag cards" in a subtle info bar (bg-wk-sky-blue border-l-2 border-wk-azure, 14px body text)
