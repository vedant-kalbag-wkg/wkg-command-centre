## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, tight inline spacing |
| sm | 8px | Button inner padding (compact), field group label gap |
| md | 16px | Default element spacing, card padding, form field gap |
| lg | 24px | Section padding, detail page section gap |
| xl | 32px | Layout gaps, page content top padding |
| 2xl | 48px | Major section breaks, empty state vertical padding |
| 3xl | 64px | Page-level top/bottom padding |

Exceptions:

- Kanban cards: 12px internal padding (between xs and md) — compact card density
- Touch targets: minimum 44px height for all interactive controls (WCAG 2.5.5)
- Table row height: 44px minimum for comfortable row selection
- Drag handles on stage management modal: 20px wide hit area — minimum accessible hit area width for a drag affordance per WCAG 2.5.5

## Typography

All typography uses `'Circular Pro', -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif` as declared in `globals.css --font-sans`.

| Role | Size | Weight | Line Height | Kerning | Usage |
|------|------|--------|-------------|---------|-------|
| Body | 14px | 400 (Book) | 1.5 | 0 | Table cell text, form labels, audit log entries, card body |
| Label | 12px | 400 (Book) | 1.4 | 0 | Column headers, badge text, metadata labels, field section headers |
| Heading | 20px | 600 (Medium) | 1.2 | 0 | Page titles (via AppShell), detail page section headings, modal titles |

Notes:
- All weights use `letter-spacing: 0`
- Uppercase is permitted only for column headers and section labels — never for body text or CTA labels

## Component-Level Interaction Contracts

### Inline Editing (Detail Pages)

- Field values display as plain text at rest
- On hover: subtle underline appears in `--color-wk-mid-grey`
- On click/focus: field transitions to editable state (input/textarea/select) with `--color-wk-azure` border
- Save: on blur OR Enter key — both triggers save to server action
- Cancel: Escape key reverts to last saved value
- Saving state: field shows spinner inline, text opacity 60%
- Save error: field border transitions to `--color-wk-destructive`, inline error message below in 12px body text
- Successful save: brief 200ms green flash on field border using `--color-wk-success`

### Table View

- Default sort: newest first (by `created_at` descending)
- Default visible columns (kiosks): Kiosk ID, Outlet Code, Venue, Region, Pipeline Stage, CMS Config Status, Install Date
- Default visible columns (locations): Name, Address, Hotel Group, Star Rating, Room Count, Kiosks Assigned
- Row hover: `--color-wk-sky-blue` (#E5F1F9) background
- Selected row (checkbox checked): `--color-wk-azure-20` background with `--color-wk-azure` checkbox
- Bulk action toolbar: fixed to bottom of viewport, slides up when 1+ rows selected, shows count + actions
- Empty filter result: shows empty state (see Copywriting below), not a blank table

### Kanban Board

- Card width: fixed 240px
- Card height: auto, minimum 80px
- Card internal padding: 12px
- Card border: 1px solid `--color-wk-mid-grey`, border-radius `var(--radius)` (0.5rem)
- Card hover: box-shadow `0 2px 8px rgba(0,0,0,0.12)`, cursor `pointer`
- Dragging card: opacity 80%, box-shadow `0 8px 24px rgba(0,0,0,0.2)`, slight scale 1.02
- Drag-over column: column background transitions to `--color-wk-azure-20`
- Column header: colored dot (pipeline stage color, 10px diameter) + stage name (14px Medium) + count badge
- When drag disabled (non-stage grouping): info banner below column headers: "Switch to stage grouping to drag cards"
- Column scroll: vertical scroll within column using ScrollArea component

### Pipeline Stage Management Modal

- Triggered from: Kanban header "Manage Stages" button and Settings > Pipeline Stages
- Stage list items: drag handle (⠿ icon) + colored dot + name (inline editable) + default badge (if default) + Edit + Delete
- Color picker: WeKnow brand presets row first (Graphite, Azure, Night Grey, Mid Grey, Sky Blue, Sea Blue, Night Blue, Pink, Gold, Green), then react-colorful hex picker below
- Delete with reassignment: inline confirmation within the modal (not a nested dialog) showing "X kiosks will be moved to:" + stage select dropdown

### File Upload (Contract Documents)

- Upload button: secondary variant, "Upload document" label
- Progress indicator: linear progress bar using shadcn Progress component, `--color-wk-azure` fill
- File list item: file icon (lucide `FileText`) + filename + size + download icon (tooltip: "Download [filename]") + delete icon (tooltip: "Remove [filename]")
- Both icons are icon-only buttons; the tooltip text is the accessible label for screen readers and keyboard users
- Upload error: red border around drop zone + error message

### Audit Log Timeline (Per-Record)

- Grouped by calendar day: sticky day header in 12px Label weight
- Entry layout: actor avatar (20px) + actor name (14px Medium) + action description + timestamp (12px muted)
- Field change: "changed [Field Name] from [old value] to [new value]" — old value in strikethrough
- Assignment change: distinct icon (lucide `ArrowRightLeft`) + "reassigned to [Venue Name]"
- Load more: text button "Load 20 more entries (N remaining)" in Azure
- Empty: "No activity recorded yet" in muted body text

### Empty States

| Context | Heading | Body |

### Error States

| Context | Message |
|---------|---------|
| Inline field save failure (generic) | "Couldn't save. Try again." (inline, below field) |
| Inline field save failure (validation) | "[Field name] [specific validation message, e.g. 'is required' / 'must be a valid date']" |
| File upload failure | "Upload failed. Check your connection and try again." |
| File upload size exceeded | "File too large. Maximum size is 10 MB." |
| Form submission failure (create/edit page) | Toast: "Something went wrong. Your changes weren't saved. Please try again." |
| Stage delete — reassignment required | Inline in modal: "Select a stage to move [N] kiosks before deleting." |
| CSV export failure | Toast: "Export failed. Try again or contact support." |
| Bulk edit failure | Toast: "Some records couldn't be updated. Please review and try again." |
| RBAC — sensitive field access denied | Inline redacted field: "Restricted — contact your admin for access." |

### Destructive Actions

| Action | Trigger | Confirmation |
|--------|---------|-------------|
| Archive kiosk | Delete button on kiosk detail page | Dialog: "Archive this kiosk?" / "This kiosk will be hidden from all views. You can restore it by filtering for archived records." / [Archive] [Cancel] |
| Archive location | Delete button on location detail page | Dialog: "Archive this location?" / "This location will be hidden from all views. Kiosks assigned here will not be affected." / [Archive] [Cancel] |
| Delete pipeline stage (no kiosks) | Delete icon in stage management modal | Inline confirmation in list item: "Delete '[Stage Name]'?" [Delete] [Cancel] |
| Delete pipeline stage (has kiosks) | Delete icon in stage management modal | Inline in modal: "Move [N] kiosks to another stage before deleting." + stage picker — no separate dialog |
| Remove contract document | Delete icon on file list item | Inline: "Remove [filename]?" [Remove] [Cancel] — no dialog, inline confirmation |
| Bulk archive | "Archive" in bulk toolbar | Dialog: "Archive [N] records?" / "These records will be hidden from default views." / [Archive] [Cancel] |

## View-Specific Layout Contracts

### Kiosks Page (`/kiosks`)

```
AppShell header: "Kiosks" title + "Add kiosk" CTA button (Azure, right)
View type tabs: [Table] [Kanban] — Table is default, no URL persistence of active tab
Table toolbar: [Search input] [Filter] [Group by] [Columns] — left aligned; [Export CSV] — right aligned
Saved views bar: horizontal scrollable pill list below toolbar; [+ Save view] at right end
Table: TanStack Table with sticky header, checkbox column (leftmost), sortable columns
Bulk toolbar: fixed bottom, appears on row selection
```

### Kiosk Detail Page (`/kiosks/[id]` and `/kiosks/new`)

```
AppShell header: Kiosk ID as title + "Archive" button (destructive, right) for existing; "Create kiosk" (Azure) for new
Tabs: [Details] [Audit]
Details tab:
  Collapsible section: Identity (kiosk ID, outlet code, customer codes)
  Collapsible section: Hardware & Software (model, serial, software version, CMS config status switch)
  Collapsible section: Deployment (install date, pipeline stage select, deployment phase tags, region, current venue + Reassign button)
    Sub-section (collapsible): Assignment History timeline
  Collapsible section: Billing (maintenance fee input, free trial switch + trial end date)
Audit tab:
  Day-grouped timeline list + Load more
```

### Locations Page (`/locations`)

```
AppShell header: "Locations" title + "Add location" CTA button (Azure, right)
View type tabs: [Table] only in Phase 2 — no Kanban for locations
Table toolbar: [Search input] [Filter] [Group by] [Columns] — left; [Export CSV] — right
Table: TanStack Table with sticky header, checkbox column
Bulk toolbar: fixed bottom, appears on row selection
```

### Location Detail Page (`/locations/[id]` and `/locations/new`)

```
AppShell header: Location name as title + "Archive" button (destructive) for existing; "Create location" (Azure) for new
Tabs: [Details] [Kiosks] [Audit]
Details tab:
  Collapsible section: Info (name, address, lat/long, star rating, room count, hotel group, sourced-by)
  Collapsible section: Key Contacts (JSONB array — add/edit/delete contact rows)
  Collapsible section: Contract (start date, end date, value, terms textarea + document file list + Upload document)
  Collapsible section: Banking (JSONB fields — redacted/locked for Viewer role)
Kiosks tab:
  Table of all kiosks assigned (current + historical) with status badges and assignment dates
Audit tab:
  Day-grouped timeline list + Load more
```

### Global Audit Log (`/settings/audit-log`)

```
AppShell header: "Audit Log" title (no CTA)
Filter bar: [User filter] [Entity type filter] [Date range picker]
Table: Date, User, Entity Type, Record Name (linked), Field, Old Value, New Value
Pagination: Load more button at bottom
```
